# v0.3.3 Desktop Host Persistence Plan

## Summary

- Goal: make the desktop host restore basket layout and host settings across restarts, so `v0.3.2` moves from demo-only layout behavior to reusable alpha behavior.
- Scope for this version is limited to desktop-host metadata persistence: basket position, snap edge, collapsed state, and host settings.
- This version must not persist scanned desktop items, must not move real desktop files, and must not change the old `workspace.json` organizer flow.

## Implementation Changes

- Add a dedicated desktop-host state model instead of extending the old workspace model.
  - `DesktopHostState` contains `DesktopHostSettings Settings` and `List<PersistedBasketLayout> BasketLayouts`.
  - `PersistedBasketLayout` contains `BasketKey`, `X`, `Y`, `SnapEdge`, and `IsCollapsed`.
- Add a stable `Key` field to `Basket`.
  - Do not use `Id` for persistence because baskets are recreated on every scan.
  - `DesktopClassifier` must emit fixed keys for default baskets: `applications`, `documents`, `folders`, `media`, `other`.
  - `Name` remains display text only.
- Add a new persistence abstraction for desktop-host state.
  - New interface: `IDesktopHostStateStore`
  - New JSON implementation: `JsonDesktopHostStateStore`
  - Storage path: `%LocalAppData%/DesktopOrganizer/desktop-host-state.json`
  - Missing file returns default `DesktopHostState` with current default settings and no saved layouts.
- Change startup flow in `App.xaml.cs`.
  - Load desktop-host state before constructing `SettingsViewModel` and `DesktopHostViewModel`.
  - Apply saved language from loaded settings before creating tray/menu text.
  - Stop constructing `DesktopHostSettings` with a hardcoded `new` default path.
- Change basket initialization in `DesktopHostViewModel`.
  - Keep current scan and classify flow.
  - After each basket is created, try to apply a saved layout by `BasketKey`.
  - If a saved layout exists, restore `X`, `Y`, `SnapEdge`, and `IsCollapsed`.
  - If no saved layout exists, use current `PlaceBasket` default placement logic.
  - After restore, clamp coordinates to the current virtual screen so old positions do not render off-screen after resolution changes.
- Add save-on-change behavior for basket layout.
  - Any change to `X`, `Y`, `SnapEdge`, or `IsCollapsed` must update in-memory host state.
  - Persist using a debounce strategy similar to `MainWindowViewModel` to avoid writing on every drag event.
  - `RefreshDesktop` must preserve layout by reapplying saved state after re-scan, so refresh does not reset user placement.
- Persist settings through the same host-state store.
  - Changes in `SettingsViewModel` update `DesktopHostState.Settings`.
  - Immediate in-session localization behavior can stay as-is.
  - The required guarantee for this version is correct restore after restart.
- Keep saved layout list aligned to currently existing baskets.
  - If a basket key disappears after re-scan, remove it from persisted state on next save.
  - If a new basket key appears, place it with default layout first, then persist from that point forward.

## Interfaces And Types

- Extend `DesktopOrganizer.Core.Models.Basket` with `string Key`.
- Add `DesktopOrganizer.Core.Models.DesktopHostState`.
- Add `DesktopOrganizer.Core.Models.PersistedBasketLayout`.
- Add `DesktopOrganizer.Core.Abstractions.IDesktopHostStateStore`.
- Add `DesktopOrganizer.Infrastructure.Persistence.JsonDesktopHostStateStore`.

## Test Plan

- Unit tests:
  - store returns default `DesktopHostState` when file does not exist
  - settings and basket layouts round-trip through JSON save/load
  - `DesktopClassifier` emits stable keys for each default basket
  - restored coordinates are clamped into the visible virtual screen range
- Manual verification:
  - drag baskets, exit from tray, relaunch, and confirm positions are restored
  - collapse a basket, exit, relaunch, and confirm collapsed state is restored
  - change language and host settings, exit, relaunch, and confirm they are restored
  - drag baskets, click refresh, and confirm existing baskets keep their saved positions
  - add or remove desktop files so basket composition changes, and confirm existing keys restore while new keys use default placement
  - force-kill the app, relaunch, and confirm desktop icon recovery still works and host-state loading does not break startup

## Assumptions

- `v0.3.3` persists host settings and basket layout together in one state file; splitting them would create two competing sources of truth.
- Basket identity is based only on classifier-defined stable keys; this version does not support user-defined basket identity.
- This version continues to use virtual-screen coordinates only; no per-monitor ownership, DPI-specific restore, or multi-monitor placement rules are added.
- Scanned desktop items remain runtime-derived data and are not persisted in this version.
