# v0.3.1

## Version overview

This version fixes the tray exit path so Desktop Organizer can fully shut down and restore desktop icons correctly.

## Added

None.

## Changed

| File | Description |
|------|-------------|
| `src/DesktopOrganizer.App/DesktopOrganizer.App.csproj` | Bumped application version metadata from `0.3.0` to `0.3.1`. |
| `src/DesktopOrganizer.App/DesktopHostWindow.xaml.cs` | Added an explicit shutdown bypass so the host window can close for a real application exit instead of always hiding to tray. |
| `src/DesktopOrganizer.App/App.xaml.cs` | Changed tray exit handling to close settings, permit real host shutdown, and then exit cleanly so desktop icon restoration can run. |
| `AGENT.md` | Restored the file to the original high-level 1-5 engineering guidelines requested by the user. |
| `docs/project-rules.md` | Moved versioning, changelog, backup, bug-tracker, WPF, encoding, naming, project-structure, and MVVM/XAML-specific project rules into a separate rules file, and added a requirement to check rule changes and the current project version at the start of each conversation. |
| `docs/bug-tracker.md` | Added the tray exit / hidden desktop icon bug entry. |

## Removed

None.

## Known Issues

- Desktop icon restoration still depends on the recovery snapshot path if the process is force-killed.
- The desktop-layer integration remains best-effort and is not yet full Fences-equivalent shell behavior.
